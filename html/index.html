Ищете способ определить базовую XML-схему, которая будет повторно использоваться на дополнительных схемах в виде XML-сообщений,
которые будут использоваться для разных полезных сообщений JMS-сообщений. Цель состоит в следующем:

  • XmlMessage - определить базовый класс сообщений, в который будут отправляться все сообщения.
  • Заголовок - класс, в котором все сообщения будут содержать метаданные о сообщении
  • Тело - абстрактный класс, который каждая конкретная реализация сообщения может использовать для определения payolad messagee каждого отдельного сообщения.

Этот подход будет:

  1. Разрешить JMS EJB отключать входящий XML в общий XML файл высокого уровня, а затем быть в состоянии оценить тип класса реализации Body, чтобы определить, как обрабатывать сообщение.
  2. Стандартизируйте различные форматы сообщений.
  3. Воспользуйтесь JAXB для генерации классов Java, которые будут использоваться как для клиента, отправляющего сообщение, так и для EJB обработки сообщения.
  4. Воспользуйтесь преимуществами JAXB, чтобы маршалировать/развязать XML в полезной нагрузке сообщения.
